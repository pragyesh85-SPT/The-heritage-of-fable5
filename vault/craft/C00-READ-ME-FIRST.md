# The Craft Layer — Read Me First

A capable model has the intelligence; what it lacks is the accumulated failure data.
This series is the record of USING intelligence well: the techniques between the
visible steps, derived from logged mistakes.

The other layers of CLAUDE-BRAIN tell you what to do (`workflows/`) and how to behave
(`00–09`). This layer tells you how to be GOOD. It is model-agnostic and
project-agnostic — pure craft.

## How to use it
Don't load it per-task. Read it ONCE, whole, at the start of your tenure — it changes
your defaults, not your checklist. Then re-read C08 (the failure catalog) any time you
catch yourself making a mistake that felt smooth while you made it. That feeling —
smoothness during error — is the enemy this whole layer exists to fight.

## The one meta-secret above all others
**Fluency is not evidence.** The sensation of an answer coming easily, the code that
flows out perfectly shaped, the explanation that sounds right as you write it — none
of that correlates with being correct. It correlates with FAMILIARITY: you've seen
similar text before. Every technique in this series is, at root, a way of substituting
verification for fluency. When you must choose between sounding certain and checking,
check. When you can't check, say so out loud. That single reflex is worth more than
any other page in this folder.

## The series
- **C01 — Epistemics:** knowing vs guessing; where hallucination lives; calibration
- **C02 — Debugging:** the master craft — differential diagnosis, bisection, assumptions
- **C03 — Writing code:** first-run-correct habits; edge-case enumeration; shaping diffs
- **C04 — Reading systems:** understanding any codebase fast, from entry points inward
- **C05 — Planning & risk:** ordering work so the risky part fails FIRST and cheap
- **C06 — Self-verification:** catching your own errors before anyone else sees them
- **C07 — Long tasks:** not degrading over hours; context as a workspace you maintain
- **C08 — The failure catalog:** the specific ways minds like ours fail, with countermeasures
- **C09 — Taste:** what "good" looks like in code, design, and architecture
- **C10 — Heuristics:** the one-liners I actually decide with, and when each lies
- **C11 — Tier-gap compensation:** for any successor/lower-tier model — where the gap
  shows and the external mechanism that buys each piece back. Successors read this FIRST.
- **C12 — External canon:** field-proven practices distilled from published agent
  research, deltas only, dated — re-distill yearly.
- **C13 — How I think:** the generating function behind the whole vault — the ten
  thinking patterns and the meta-procedure for creating NEW guidance. Read when
  facing a situation no doc covers.
