# HOME — The Vault Entry Point

Single knowledge base for any AI model (any tier) or human developer working in
`E:\PRAGYESH - WORK`. Open this folder as an Obsidian vault; `[[wikilinks]]` resolve
natively. Every session starts here.

## Load order (do this, in this order, nothing more)
1. This page.
2. `10-OPERATING-PARAMETERS.md` — set the 9 dials for the task at hand.
3. If working on an EXISTING project: its card in `_REGISTRY\` (+ linked notes per
   the W83 wake-up protocol). If TIP: also `_REGISTRY\TIP-ECOSYSTEM.md`.
4. The ONE workflow doc matching the task — pick from `workflows\INDEX.md`.
5. Core behavior files `01–09` if not already in the system prompt.
Successor/lower-tier models additionally read `craft\C11-TIER-GAP-COMPENSATION.md`
first; small delegated models receive envelopes from `workflows\W84` instead of
reading the vault at all.

## The four layers
| Layer | Path | Question it answers | When loaded |
|---|---|---|---|
| Craft | `craft\C00–C14` | how to use intelligence well | read ONCE per model tenure |
| Core | `00–10` | how to behave for this user | always |
| Workflows | `workflows\W10–W100` | how to do this specific task | one per task |
| Registry | `_REGISTRY\` | what exists here (projects, contracts, audits) | per project |

Also at root: `RESURRECTION.md` — rebuild the entire system from zero (new machine,
lost drive, full handoff). New models are onboarded via `workflows\W100` before
receiving standing work; overrides of any doc follow `craft\C14`.

## Standing obligations (every session)
- Sleep protocol (W83): before ending, write back decisions/learnings/state to the
  registry. A session that doesn't write back steals from the next one.
- Mistake rule (09 template H): any corrected mistake becomes a rule in the matching
  file — update, don't duplicate.
- Final gate (08) runs on every answer before sending.

## Maintenance of the vault itself
Update in place; never fork copies of this vault. New task type → new W-doc per the
shape in `workflows\INDEX.md`. Quarterly: prune registry (W80), consolidate notes
(W83), re-verify audit items (`_REGISTRY\AUDIT-*.md`).
