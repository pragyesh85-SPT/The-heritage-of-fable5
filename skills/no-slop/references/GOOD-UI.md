# GOOD-UI — What Good Is Actually Made Of

Good UI is not a style; it's FIT — the design shaped by this product's content,
genre, and audience until it couldn't belong to anyone else. The craft below is
what the consistently-praised products (the Linears, Stripes, Airbnbs of any
era) share UNDER their different skins.

## The five properties every good UI shares
1. **Instant hierarchy** — one glance answers: what is this, what matters most,
   what do I do. Built from size/weight/spacing, not decoration.
2. **Earned density** — exactly as much information as the user's task needs.
   Operators get more, browsers get less. Wrong density = wrong tool feeling.
3. **A voice** — type, color, and copy sound like ONE brand with an opinion.
   You could identify the product from a cropped screenshot.
4. **Motion discipline** — the UI feels fast because nothing moves without a
   reason and every action is acknowledged <100ms.
5. **Designed edges** — empty, loading, error, long-content, and first-run
   states look intentional. Amateur UIs are good at the happy path only.

## Genre archetypes (study the pattern, never copy the skin)
- **SaaS/dev-tool landing** (Linear, Stripe class): near-monochrome + ONE
  accent; the PRODUCT ITSELF is the hero image; short declarative headlines
  ("what it does" in ≤8 words); generous but rhythmic spacing; real UI
  screenshots over illustrations.
- **Commerce/brand** (premium D2C class): photography carries the design —
  real product, real process, real people; warm editorial type; story sections
  with few words; trust strip (origin, process, policies) near the buy action.
  For TIP: the ghee, the Athpaheri process, the family — not stock cows.
- **Operator/admin** (broker back-office class): tables with sticky headers,
  row density ~40–48px, inline filters, bulk actions, keyboard reachability;
  color reserved for status; zero marketing air.
- **Field PWA** (delivery-app class): one primary action per screen, 44px+
  targets, bottom-reachable controls, high contrast for sunlight, forgiving
  offline behavior.

## The token recipe (numbers that work; deviate on purpose only)
- **Type:** one family for UI (a neutral grotesk) + optionally one display
  family for brand moments. Scale ~1.25 ratio: 12/14/16/20/25/31/39. Body 16,
  line-height 1.5, max 70ch. Weights: 400/500/650 — three, not seven.
- **Color:** 1 accent + 9-step neutral ramp + semantic green/red/amber. Text
  #1a1a1a-class on #fafafa-class (never pure-on-pure). Accent used on <10% of
  any screen — scarcity is what makes it mean "act here".
- **Contrast floors:** 4.5:1 body, 3:1 large text/icons — non-negotiable.
- **Spacing:** 4-based scale; related elements ≤8 apart, groups ≥24 apart.
  When a layout feels off, fix spacing FIRST — it's the cause 70% of the time.
- **Radius:** one token (4–8px business, 10–14px consumer). **Shadows:** two
  levels max (raised, overlay).
- **Motion:** 150–250ms, ease-out, transform/opacity only.

## Composition rules
- Design the most important screen state FIRST at real density (not the empty
  marketing shell) — layouts that only work with 3 items are broken with 30.
- Left-align body text; center only short display lines.
- One primary button per view; everything else is secondary/ghost.
- Icons accompany labels, never replace them (except universally known: ×, ⚙).
- Images: real product/real people or none. A missing image beats a stock one.

## How to STUDY a good site (the kimi-webbridge routine, token-cheap)
1. Pick the reference (SKILL gate 3). 2. Via kimi-webbridge get: computed body
font-size/line-height/family, heading sizes, the accent color value + where it
appears, spacing between sections vs within them, nav item count. 3. Write those
numbers down as "reference DNA" next to your tokens. 4. Copy the DISCIPLINE
(scale, restraint, density), never the palette or logo-adjacent styling.

## The one-line summary
Slop is what happens when nothing says no. Good UI is a stack of deliberate
no's — one accent not five, one radius not four, one primary action not three —
applied to real content for a named audience.
